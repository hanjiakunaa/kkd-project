# Git 自动提交工具 - 使用说明 🚀

## 🎯 一键提交

```bash
# 最简单 - 自动生成提交信息
npm run push
npm run quick
npm run git
npm run commit

# 带自定义提交信息
npm run push "你的提交信息"
```

> 💡 四个命令效果完全相同，选你喜欢的用！

## ✨ 自动生成的提交信息

根据文件变更类型自动生成：

- `✨ 新增功能 (3个文件) - 2025-11-02 14:30:25`
- `🔧 更新代码 (5个文件) - 2025-11-02 14:30:25`
- `📦 代码更新 (新增2, 修改3个文件) - 2025-11-02 14:30:25`

## 🔄 工作流程

```
修改代码 → npm run push → 完成！
```

自动执行：

1. ✅ 检查 Git 仓库
2. ✅ 检查文件变更
3. ✅ git add .
4. ✅ git commit
5. ✅ git push

## 💡 使用场景

```bash
# 日常开发
npm run quick

# 功能开发
npm run push "完成用户管理功能"

# Bug 修复
npm run git "修复登录页面样式 #123"

# 文档更新
npm run commit "更新 API 文档"
```

## ⚠️ 注意事项

- ✅ 没有变更时自动退出
- ✅ 自动设置上游分支（首次推送）
- ✅ 推送失败会有详细提示

## 🐛 常见问题

**推送失败？**

```bash
# 检查网络和远程仓库
git remote -v
```

**撤销提交？**

```bash
# 撤销最后一次提交（保留更改）
git reset --soft HEAD~1
```
